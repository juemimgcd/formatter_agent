AI Agent 开发面试要求

一、面试题目

请基于任意 AI Agent / LLM 应用框架，完成一个从自然语言输入，到联网搜索，到结构化整理，再到生成 Excel 文件的完整工程。

候选人可自由选择技术栈，例如但不限于：

LangChain
LangGraph
OpenAI / GPT SDK
AutoGen
CrewAI
LlamaIndex
其他自选 Agent 框架

二、题目背景

企业内部很多场景并不是简单对话，而是需要 AI 代理完成一整条任务链路，包括：

理解用户输入
制定执行步骤
调用外部搜索能力获取信息
对结果进行提取、去重、归纳和结构化
输出标准化结果文件，供业务人员继续使用

本题希望考察候选人是否具备构建 可用、可扩展、可观测 的 AI Agent 工程能力。



三、任务要求
任务描述

请开发一个 AI Agent 应用。用户通过 Chat 输入一个关键词或主题，例如：

“简历”
“Java 后端工程师简历”
“上海 AI 产品经理简历模板”
“海外销售岗位简历范例”

系统需要自动完成以下流程：

解析用户意图
识别用户要搜索的主题
判断搜索目标和输出字段
联网搜索
从互联网获取相关内容
可使用搜索引擎 API、公开网页抓取、RAG 检索等方式
信息抽取与整理
从搜索结果中提取有效信息
对重复、低质量、无关内容进行过滤
将结果整理为结构化数据
生成 Excel
将整理后的结果输出为 Excel 文件
Excel 中至少包含多个字段列
字段需具备业务意义，不可只是简单把网页内容原样堆进去
结果反馈
在 Chat 中返回任务执行结果摘要
提示 Excel 文件生成成功
可附带部分预览数据
四、示例场景
示例输入

“简历”

示例预期行为

系统自动执行：

将“简历”识别为搜索主题
去网上搜索相关简历样本、模板或候选人资料页面
提取结构化字段
输出 Excel 文件，例如 resume\_search\_result.xlsx
Excel 示例字段

注：如果真实网页中无法稳定提取“姓名/学历/工作年限”，可以根据数据来源重新设计字段，但必须说明字段设计理由。



五、工程要求

候选人提交的内容应尽量体现真实工程能力，而不是一次性脚本。

建议工程结构

至少应包含：

README
核心代码
配置文件
依赖说明
运行方式
示例输出文件
代码要求
代码结构清晰
模块划分合理
具备基本注释
配置与业务逻辑尽量分离
不允许将密钥硬编码在代码中
应说明使用的模型、搜索方式和 Excel 生成方式
推荐模块划分

例如：

chat\_input
planner
search\_tool
content\_parser
llm\_extractor
deduplicator
excel\_exporter
logger

七、交付物要求

候选人需要提交以下内容：

1. 源码工程
可运行的完整代码仓库
2. README 文档
README 中至少说明：

项目简介
技术栈选型
系统架构说明
核心执行流程
环境准备
启动方式
示例输入输出
已知问题与优化方向

3. 示例运行结果

至少提供：

一个示例输入
任务执行日志或截图
一个生成好的 Excel 文件

4. 架构说明

建议附一张简单流程图，说明：

User Chat -> Agent -> Search Tool -> Content Extract -> Structured Data -> Excel Export

八、评分标准

总分 100 分，可参考如下维度：

1. 需求理解与方案设计（20 分）
是否正确理解题目目标
是否有合理的 Agent 分层设计
是否考虑输入、搜索、抽取、输出的完整链路
2. 工程实现能力（25 分）
代码结构是否清晰
模块划分是否合理
是否具备可运行性和可维护性
是否体现真实工程习惯
3. AI Agent 能力设计（20 分）
是否真正使用了 Agent / Tool / 多步推理机制
是否不是简单单次 prompt 拼接
是否体现任务规划、工具调用、状态流转等能力
4. 数据处理质量（15 分）
搜索结果是否有效
抽取字段是否合理
是否做了过滤、去重、清洗
输出 Excel 是否具备可读性
5. 异常处理与鲁棒性（10 分）
搜索失败时如何处理
页面解析失败如何兜底
内容为空时如何反馈
是否有日志和错误提示
6. 文档与表达（10 分）
README 是否清晰
技术说明是否完整
是否能让评审快速理解和运行项目
